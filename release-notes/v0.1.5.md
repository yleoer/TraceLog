# TraceLog v0.1.5

新增 Tempo Time 记录、独立更新助手和 Jira 刷新能力，并强化服务可靠性与内部结构。

## 新增

- 添加 Tempo Time 记录功能，支持选择工作项、日期范围、工时和说明后批量写入 Tempo。
- 新增时间记录页面和导航入口，并在设置中支持 Tempo/Jira/AI 相关配置。
- 新增独立更新助手，用于下载、校验并替换应用安装包。
- 新增 Windows NSIS 安装器配置。
- Issue 详情页新增 Jira 刷新按钮，可从 Jira 拉取最新 Issue 信息。

## 优化

- 将服务层拆分为 issues、temp_tasks、weekly、time、uploads、integrations 等领域服务。
- 将宽 Repository 拆分为多个领域小接口，降低服务和测试替身耦合。
- 将上传、Markdown、Jira、时间、搜索索引等 helper 拆入独立文件，降低单文件复杂度。
- 修复 Release workflow 的资产上传流程。

## 可靠性

- Store 写操作增加事务边界，确保实体、活动和搜索索引一起提交或回滚。
- 桌面入口增加 context timeout，避免长时间调用无限挂起。
- 上传增加大小限制和 base64 预检。
- JSON 扫描错误不再静默吞掉，便于定位脏数据。
- `weekRange` 增加 ISO week 合法性校验。
- 增加 service、store、desktop 和 updater 相关单元测试。
