# TraceLog v0.1.7

修复在线升级助手启动失败问题，并为更新流程增加完整的持久化诊断日志。

## 修复

- 升级时复用最近一次版本检查结果，避免点击升级后重复请求 GitHub。
- 更新助手使用唯一临时文件启动，避免残留文件或进程 ID 复用导致冲突。
- 更新助手子进程启动后主动释放进程句柄，提升独立运行稳定性。
- 升级未启动时在界面显示明确错误，不再把失败结果显示为成功。
- 更新失败信息增加具体原因、手动下载提示和升级日志路径。

## 诊断

- 将升级日志持久化到应用数据目录下的 `logs/update.log`，并以追加模式保留多次升级记录。
- 使用会话 ID 串联桌面进程与独立更新助手的同一次升级过程。
- 记录版本检查、GitHub 响应、资产选择、助手复制与启动、下载安装包、SHA256 校验、等待主程序退出、安装器执行、文件清理和应用重启结果。
- 日志包含组件名、UTC 微秒时间戳、进程 ID、耗时和错误详情，不记录 Jira、AI 等配置密钥。
